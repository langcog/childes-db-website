Releases

childes-db will be updated every 6 months with a most recent parse of the CHILDES database, reflecting the addition of new corpora and the revision of older annotations (more details can be found in the documentation). Visualizations and the childesr package will always use the most recent database version by default. The primary method of interacting with childes-db, the R API childesr, can be directed to use any recent database version by using the --version flag. We provide here compressed .sql files for users wanting to refer to older releases or to install the database locally.

  • childes-db-version-0.1.0

To use the .sql file you will need to have a working version of MySQL server and a client. We recommend MAMP and SequelPro as easy-to-use server and client software. Loading the database from the .sql file is fastest and easiest from the command line client. Remember to replace the name of the .sql file with the filename you downloaded.

  1. If your browser did not automatically unzip the .sql.gz file, unzip it with gunzip childes-db-version-0.1.0.sql.gz
  2. Then from the same directory with the uncompressed .sql file, connect to MySQL: mysql --host=127.0.0.1 --port 8889 --user=root --password=root (assumes default MAMP credentials)
  3. From within the MySQL console, create the database: create database childesdb-0.1.0
  4. Quit the MySQL console with \q
  5. Load the sql dump into the database with mysql --host=127.0.0.1 --port 8889 --user=root --password=root childesdb-0.1.0 < childes-db-version-0.1.0.sql
    6. (this will take several minutes)
  6. refer to the documentation for childesr regarding how to use a local database server
